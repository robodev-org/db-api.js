# Contributing

## Purpose

Define contribution expectations for the documentation and code.

## Scope

Covers workflow, diagnostics-first tests, and review expectations. Does not replace root CONTRIBUTING.md.

## Table of Contents
{:toc}

## Workflow

- Open an issue describing the change.
- Submit a PR with tests and diagnostics evidence.
- Update relevant docs and defaults tables.

## Diagnostics-first tests

- New behavior MUST include diagnostics evidence.
- Tests MUST fail if diagnostics are missing.

## Required updates

- API changes MUST update API docs and README defaults.
- Diagnostics changes MUST update Diagnostics Model docs.
