# Reproduction Guides

## Purpose

Provide a stable process for reproducing issues with diagnostics evidence.

## Scope

Covers minimal reproduction steps and required artifacts. Does not cover backend debugging.

## Table of Contents
{:toc}

## Minimal reproduction

- Isolate a single repository call.
- Enable diagnostics.
- Record the DiagnosticsReport and attach it to the issue.

## Required artifacts

- DiagnosticsReport JSON
- SDK version and options
- Table name and select fields

## Optional artifacts

- Cache key parameters
- Call trace sequence from tests
