# Reporting Security Issues

## Purpose

Explain how to report security issues safely.

## Scope

Covers reporting channel and required information. Does not cover issue triage policy.

## Table of Contents
{:toc}

## Reporting process

- Report issues privately to maintainers.
- Provide steps to reproduce and impact assessment.
- Include a diagnostics report if relevant.

## Response expectations

- Acknowledgement within 72 hours.
- Mitigation plan provided after triage.
